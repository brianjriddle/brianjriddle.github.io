26 Nov 2015

Jag fick frågan har om dagen om j2ee mina tänker omkring detta. Jag måste erkänna att när jag började fundera på j2ee kom jag på hur lite jag faktiskt använder av detta alls nu förtiden. Ut över webapplikationer det är inte mycket tack vare spring och de tools som har vuxit fram de senaste 5 år.

Hur som helst here it goes.

Jag jobbar med “j2ee” dagligen men använder en extremt liten del av “j2ee” hos min nuvarande arbetsgivare. Det delen vi använder av oss mest (utöver spring för dependency injection) är Spring MVC och lite av Springs JDBCTemplate.

Detta är den 4:e arbetsplats som har haft java som programmering språk och den är den förste som har ganska bra tänke på hur man ska bygger saker. Visst finns en del äldre kod som är inte lika väcker men allt kod som är skriven sedan nästan 4 år tillbaka ta mycket inspiration från Joshua Bloch’s Effective Java 2nd edition.

Särskilt den delen angående hur man skriver immutable kod via några design patterns som “The Builder pattern”. Det tog lite tid för mig att sätta mig i detta och det var mycket annorlunda än hur jag jobbade med java tidigare men vinsten med att slippa en del trådning problem är värt detta. En annan grej som är annorlunda är att det är mycket mindre ärv som vi använder. Väldigt sparsamt ska jag säga.

Det som är kanske den är där vi har börjat att driva utveckling senaste tiden är mot microservices. Vi använder spring boot för det vi gör men det finns dropwizard. Båda bygger man binär och sedan kör man som det är.

Vi som många andra har börjat att använder angular som presentations teknik ansluten till vår microservices.

Sedan mitten av oktober har vi använt Java 8 i alla projekt vi har. Det känns mycket roligare än det som java har varit den senaste 5-6 år. Även om vissa av saker på minne man om hur det var när generics kom i java 5. Kull i början men kan spåra ut lätt…

Hur som helst venkat subramaniam’s book om functional programming är en bra överblick av functional programming i java idag. Denna sida är ganska bra om vad som har hant mellan java 7/8.

Nedan är en link samling som bord har en bra start punkt om lite mer modern java. Det finns mer att skriva men om junit, maven, gradle,Saas till exampel heroku, continous integration, versionhantering, osv… men det är det kommer man i kontakt med så fort man börja att kolla några av dessa länkar.
