24 Feb 2010

must be synchronicity but with in the last couple of days i have seen some new/old build tools for java. The first is jruby’s new rake and ant integration seen on engine yard’s blog. If you are using ant and jruby this looks like you can get the best of both worlds as you can call ant from rake and rake from ant. All with out getting swept away….

The other build tool seen was when attended a presentation tonight on buildr given by Anders Janmyr at jayway’s office. buildr is a ruby based build system for java. It is trying to create a new build system for java that does not have the baggage from ant or maven. It can use maven repos to retrieve dependency but needs to be explicitly told to retrieve transitive dependencies. This could be good could be bad.could keep you from having 3 copies of some libraries lying around. buildr does have a nice syntax and ability to point out an archive be it .zip, .tar, .tgz and retrieve a library somewhere there in, cache it locally (somewhere under ~/.m2).

Given that jruby is going to have ant integration with the upcoming 1.5 release it makes it hands down the much easier to install. Given both need a jvm the install requirements are a little more cumbersome for buildr as it can use a variety of methods to to install.
